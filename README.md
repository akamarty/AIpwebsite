# Sophia — your personal AI advisor for Porsche buyers

## Overview
**Tagline: "Know the car before you buy it."** Positioning: not another marketplace — the decision layer on top of every marketplace. Paste a VIN or a listing link, get a full buying report in 15–30 seconds: fair market value, collector desirability, spec rarity, comparable sales, ownership costs, common issues, negotiation strategy, and appreciation outlook. Starting with the Porsche market. Free users get a real, functional basic report (VIN structural decode + option-code lookup); paid subscribers unlock the rest of the report plus saved searches, portfolio tracking, alerts, and advanced analytics — a SaaS model, since enthusiasts revisit a car throughout the ownership lifecycle, not just while shopping.

Today, two of those eight report dimensions are real (the VIN structural decode and the option-code lookup); the other six (fair market value, collector desirability, spec rarity, comparable sales, ownership costs, common issues, negotiation strategy, appreciation outlook) are shown as locked/blurred Phase 2+ teasers with illustrative placeholder values, gated behind a (currently non-functional, click-through-only) free-account signup. Listing-link input is accepted and routed, but analysis of the link itself isn't built yet — it shows an honest "coming soon" state with a VIN fallback, rather than faking a scrape.

## It actually functions
This is no longer just a click-through design prototype — the decode and lookup logic is real and runs entirely in the browser, no backend required:
- **VIN decoding** (`vin-decoder.js`, `window.PorscheVIN.decodeVin`) — structural decode tables (year codes, body type, plant, chassis/model code) sourced from the [Stuttcars Porsche VIN Decoder guide](https://www.stuttcars.com/porsche-data/porsche-vin-decoder/), USA/Canada 17-character VIN format, model years 1981–present. A VIN encodes chassis/generation, model year, body type, plant, and serial — it does **not** encode individual factory-installed options; that requires a separate lookup (below).
- **Option-code lookup** (`window.PorscheVIN.lookupOption`) — 776 factory option codes extracted from the user-supplied Rennlist "Options.pdf" reference (Porsche option codes, model years 1964–2003, compiled by Adrian Streather), shipped as `porsche-options.json` (generated from `porsche-options.xlsx`, which documents full provenance and category breakdown in its own source note). Some codes were reused by Porsche across different years for unrelated options — where the source lists multiple meanings for one code, all are shown separated by " | ".
- **No search history.** VIN Results doesn't log or display past searches — removed per explicit request so visitors never see other queries on the device (it was always local-only via `localStorage`, but the feature is gone rather than just hidden).
- **Shared VIN knowledge base + auction history — now backed by a real database.** A Supabase (Postgres) project is live (see `schema/`) with three tables: `vin_records` (the Step 1 knowledge base — VIN/Year/Model/Trim/Transmission/Color/Interior/Engine/Options/Production market, with provenance tracking), `porsche_options` (mirrors the 776-code reference, already seeded), and `auction_sales` (Phase 2, schema only — ships empty on purpose, see `schema/DATABASE_SCHEMA.md` for why). This was a deliberate architecture change from the original "client-side only" decision: a knowledge base other users contribute to can't live in one visitor's browser. The frontend isn't wired to this yet (no submission form, no live comps query) — that's the next piece of work.
- **718 production & color-rarity data — real numbers, both in the database and shipped statically.** Two more Supabase tables, `model_allocations` and `color_distribution` (`schema/003_production_market_data.sql`), hold real user-supplied production counts and factory exterior-color breakdowns for the 982-generation 718 lineup (Spyder, Cayman GTS 4.0, Boxster GTS 4.0, Boxster 25 Years, Cayman GT4, Spyder RS). The same numbers are mirrored statically in `market-data.js` (`window.SophiaMarketData`) so the Market Data page and VIN Results' 718 callout render instantly — same reasoning as `porsche-options.json` shadowing `porsche_options`. Nothing is estimated: every count is exactly as supplied, and the Spyder/GT4 color totals cross-check exactly against their published production totals (2,054 and 2,664 respectively).
- **Real 718 Spyder factory build sheets — VIN Results shows the actual order.** `spyder_builds` and `spyder_option_reference` (`schema/004_spyder_builds.sql`, mirrored statically in `spyder-builds.js` / `window.SophiaSpyderBuilds`) hold 151 individually-verified 2020-model-year Spyder VINs (real MSRP, commission, production date, factory option codes) sourced from a user-shared spreadsheet, plus 21 confirmed option codes with description/price/take-rate. When a decoded VIN matches one of these 151 records exactly, VIN Results shows an "Official factory build sheet" panel styled after the source sheet's own "Print It" build-sheet view — real data only, never fabricated for VINs outside this batch. See `schema/DATABASE_SCHEMA.md` for what this batch covers vs. the source spreadsheet's larger (JS-tab-driven, not fully reachable this session) database.

## About the Design Files
The files are **Design Components** — self-contained `.dc.html` files with inline styles and a small runtime, `support.js` — that double as the actual functioning front end (there's no separate "real" implementation to recreate; `index.html` and `VinResults.dc.html` run the logic above directly in-browser). `CreateAccount.dc.html` and the Collector Intelligence lock screens on VIN Results remain click-through-only Phase 2+ teasers (no real auth or paid data behind them yet).

## Fidelity
**High-fidelity, and functional for VIN decode + option lookup.** Colors, type, spacing, and copy reflect the current live design. No car photography anywhere on the site — Wikimedia Commons coverage never included the 718 lineup, so the old chassis-photo panel on VIN Results broke for most real-world VINs; it's been replaced with a real, evidence-based AI Summary (see `ai-summary.js`) rather than patched with more placeholder slots.

## Design System
"Modernist": flat, architectural, mono red-on-white, Archivo typeface, 0 border-radius, strong 2px dividers. Tokens are CSS custom properties in `styles.css` — reuse them as your Tailwind theme extension (`--color-bg #f3f2f2`, `--color-text #201e1d`, `--color-accent #ec3013`, plus 100–900 tonal ramps for neutral/accent). Component classes (`.btn`, `.card`, `.tag`, `.table`, `.nav`, `.field`/`.input`, `.dialog`) are in the same file — port these to your component library.

## Screens
1. **Homepage** (`index.html`) — minimal, single-purpose "ask" UI, no top nav: Sophia logo + name, tagline, beta banner ("full support for 997 & 718, more chassis coming soon"), gradient-accented search pill (VIN, option code, or listing link), three quick-start chips ("Spec rarity" routes to a live demo report; "Market dashboard" routes to the real Market Data page; "Opportunity alerts" routes to Pricing, a Pro/Collector-tier feature, not a fabricated page), a real data-viz hero chart (every cataloged 718 option's take rate, sorted rare-to-common, computed client-side from `spyder-builds.js`), a **Spotter's Guide** bar-chart ranking (the 10 lowest take-rate cataloged options, each bar a real factory code with a real build count out of 1,067 — no flashcard copy), and a low-key "Log in · Sign up free" text line at the bottom — no button chrome. Supporting marketing content (hero photo, feature showcase, pricing teaser, footer) is still in the file but hidden (`#more-content`) per a deliberate simplification — not deleted, just off by default.
2. **VIN Results** (`VinResults.dc.html`) — the actual "report" page. Renders the real decode of the `vin` or `code` query param (spec table, or matched option description), or an honest "listing-link analysis is coming soon" state for the `url` param. The top-right column shows a real factory build sheet (VIN, total retail price, commission, production date, country, base/delivery/options-total/MSRP breakdown) when this exact VIN matches one of the 151 verified 718 Spyder records, falling back to the plain VIN-structure table otherwise; the itemized option-code table sits full-width below it. When the decoded chassis is a 718 (982-generation), a **718 production & color rarity** section shows real GT4/Spyder/Spyder RS production and rarest-color figures, free, with a link to the full Market Data page. "Your buying report" is fully open, no account gate: Spec Rarity and Comparable Sales are computed from real data (a percentile against the 151 verified builds; an honest "0 on file" for transaction comps), the other six dimensions (fair market value, collector desirability, ownership costs, common issues, negotiation strategy, appreciation outlook) are labeled "Coming soon" rather than shown as invented numbers. An **AI Summary** panel sits where chassis photos used to: a confidence score built from real evidence factors — VIN decode, build-sheet match, trim rarity data on file, option-resolution rate, sample size behind any shown rate — plus a plain-language read of the decoded VIN and, when a build sheet matches, that car's rarest priced options. No search history is logged or shown.
3. **Market Data** (`MarketData.dc.html`) — real production & color-rarity charts for the 982-generation 718 lineup, styled after competitor RennPulse's market-data presentation (clean horizontal bar charts, a live-data summary table with rarest/most-common callouts). Sections: production by trim (bar chart), Spyder RS launch allocation (NA/RoW/Global stat cards + Weissach take rate), GT4 and Spyder color-distribution bar charts with rarest/most-common callouts, and a market-summary table. All figures sourced from `market-data.js`, shown exactly as supplied.
4. **Create Account** (`CreateAccount.dc.html`) — split screen: left accent-field panel listing unlocked report benefits, right form (Google/Apple/email+password). Still a click-through teaser — no real auth wired up.
5. **Strategy Docs** (`StrategyDocs.dc.html`) — flowing document: PRD, user flow, IA, DB schema, API architecture, SEO approach, Phase 2–6 roadmap table. Not yet updated to the new "buying assistant" positioning — still describes the earlier VIN-decoder-first framing.
6. **Pricing** (`Pricing.dc.html`) — Free / Pro ($9.99/mo) / Collector ($19.99/mo) tier cards, full feature-comparison table, FAQ (now including where the Spec Rarity data comes from, linking to Market Data), mapped to the new report dimensions: Free gets the basic report (VIN decode + option lookup); Pro adds fair market value, collector desirability, spec rarity, comparable sales; Collector adds ownership costs, common issues, negotiation strategy, appreciation outlook, plus saved searches, portfolio tracking, alerts, and monthly reports.

## Interactions & Behavior
- Homepage chips and search pill route to `VinResults.dc.html?vin=...`, `?code=...`, or `?url=...` depending on whether the input looks like a VIN, an option code, or a URL; VIN/code route to real decode/lookup, URL routes to the coming-soon state.
- VIN Results "Unlock the full report" button and locked cards route to Create Account (still a teaser, no real auth).
- Create Account submit routes back to VIN Results (post-auth landing).
- The "Garage" nav destination and save-to-Garage CTA have been removed site-wide.
- All buttons/links use the design system's themed hover/active/focus states (accent ramp steps, 2px accent focus ring) — no browser defaults.

## State Management
- `vin`/search-input state on Homepage; `mode`/`data`/`lookupCode` state on VIN Results, driven by the `vin`/`code` URL query params.
- Auth state (anonymous vs. logged-in) gates Collector Intelligence content on VIN Results — anonymous sees blurred/locked, logged-in sees real values (still a Phase 2+ teaser, no real auth yet).
- No accounts, no server, no database — see "It actually functions" above.

## Design Tokens
See `styles.css` — all colors, spacing (`--space-1..8`), radius (`0` throughout), and shadows are CSS variables there. Font: Archivo (Google Fonts), weight 800 for headings.

## Assets
VIN Results no longer shows chassis photos — Wikimedia Commons coverage only extended to ~10 curated 911-family chassis codes, so most queries (every 718 included) hit a broken "photo coming soon" slot. That panel is now an AI Summary instead: a real, evidence-based confidence score plus a spec read, computed entirely from decoded VIN fields and Sophia's own verified data (see `ai-summary.js`). Icons are inline Lucide-style SVGs (checkmarks on Create Account).

- `styles.css` — the Modernist design system (tokens + component classes) plus the Apple-Intelligence-style gradient accent layer (`--gradient-ai` tokens, gradient bar/pills/chips).

Strategy doc content (PRD/schema/API/roadmap) is also fully written out inside `StrategyDocs.dc.html` — read it directly for the DB schema and API route list. Note the schema there describes a future server-backed Phase 2+ design; the site as it stands today uses no server or database.
